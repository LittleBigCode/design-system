Console layout

The full-page application shell: a grouped sidebar nav, a ⌘K command palette, an optional Light/Dark/Sepia theme switcher, and a user avatar with sign-out. The React <ConsoleLayout> takes one nav config and drives the sidebar, the active highlight, and — by default — the palette from it; your routed content is the children.

Live demo

Because <ConsoleLayout> owns the whole viewport, there is no in-page preview here — the entire demo app is a console layout. Open it to drive the sidebar, ⌘K palette, and theme switcher live.

Props

All props are optional. nav is the single source of truth: the sidebar buttons, the active highlight, and the default ⌘K commands are all derived from it.

PropTypeNotes
brand ReactNode | { name?, sub? } Wordmark element, or { name, sub } to build one.
nav ConsoleNavGroup[] Grouped sidebar navigation. Each group is { group?, items }; each item is { id, label, badge? }.
active string Active item id — highlights the matching nav button.
onNavigate (id: string) => void Fired with the chosen item id. Point it at your router.
commands Command[] ⌘K commands. Defaults to one “Go to” entry per nav item.
user ConsoleUser Right-aligned avatar + sign-out: { initials?, name?, onSignOut?, signOutLabel? }.
actions ReactNode Extra app-bar action elements (e.g. a primary button, a status badge).
search boolean Show the ⌘K search trigger + command palette. Defaults to true. Pair with searchPlaceholder.
themes boolean Render a Light/Dark/Sepia switcher that drives data-theme on <html>.
children ReactNode The routed page content rendered in the main column.

className and searchPlaceholder round out the props — see react/components/ConsoleLayout.d.ts for the full types.

React

Define a grouped nav once, then derive the ⌘K commands from its flattened items (plus any app actions). Wire active/onNavigate to your router.

<ConsoleLayout> 
import { ConsoleLayout, Badge, Button } from "@diametral/design-system/react";

const NAV = [
  { group: "Work", items: [
    { id: "overview", label: "Overview" },
    { id: "projects", label: "Projects" },
    { id: "reports",  label: "Reports" },
  ] },
  { group: "Talent", items: [
    { id: "candidates", label: "Candidates", badge: 7 },
    { id: "team",       label: "Team" },
  ] },
];

const FLAT = NAV.flatMap((g) => g.items);

function Shell({ route, navigate, onSignOut, children }) {
  const commands = [
    ...FLAT.map((n) => ({ id: n.id, label: n.label, group: "Go to", onRun: () => navigate(n.id) })),
    { id: "signout", label: "Sign out", group: "Actions", onRun: onSignOut },
  ];
  return (
    <ConsoleLayout
      brand={{ name: "Diametral", sub: "Console" }}
      nav={NAV}
      active={route}
      onNavigate={navigate}
      commands={commands}
      user={{ initials: "VD", name: "Vincent", onSignOut }}
      themes
      searchPlaceholder="Search projects, people…"
      actions={<><Badge variant="accent">Production</Badge>
                 <Button variant="primary" onClick={() => navigate("projects")}>New</Button></>}
    >
      {children}
    </ConsoleLayout>
  );
}

ConsoleLayout is a full-page shell — render exactly one, at the root of the authenticated app. Leave commands off to get the default “Go to” palette for free. See the App shell recipe for the end-to-end pattern.
